# Contributing to Wozzits

Thank you for your interest in contributing to Wozzits.

Wozzits is a personal experimental game engine project. Contributions are welcome, especially from people who are interested in the engine, want to understand how it works, or want to add features they personally intend to use.

## Contributions are voluntary

Unless explicitly agreed in advance, contributions to this repository are voluntary and unpaid.

There is currently no budget for paid pull requests, paid issue work, bug bounties, or unsolicited contract work. Please do not submit work with an expectation of payment unless we have discussed and agreed to that arrangement before the work begins.

If paid work is ever considered, it must be scoped and agreed to in advance, including:

- the task
- expected deliverables
- acceptance criteria
- payment amount
- payment method

A merged pull request does not imply payment.

## AI-assisted contributions

AI-assisted contributions are encouraged. Pull requests should be accompanied by appropriate tests.
Place tests in tests/<appriopriate test category subfolder>/new_test.cpp

## Preferred contributions

Large architectural changes should be discussed first.


## Working on issues

If you want to work on an open issue, feel free to comment on the issue first.


## Code style and project direction

Wozzits is evolving quickly. Some parts of the engine are experimental, and some are intentionally simple while the design is being explored.

Please favor:

- clear code over clever code
- small changes over large rewrites
- tests over assumptions
- explicit ownership and lifetime rules
- deterministic behavior where possible
- preserving existing design boundaries

The project is especially interested in rendering, scene compilation, asset pipelines, diagnostics, tooling, and engine architecture.

## No spam or drive-by automation

Please do not use this repository as a target for automated contribution farming.

Unwanted examples include:

- mass-generated typo PRs
- unrelated formatting churn
- dependency bumps without testing
- generic README rewrites
- changes that appear to be made only to solicit payment
- PRs opened without understanding the project

Contributions should be made in good faith and should improve the project.

## License and contribution terms

By submitting a pull request, you confirm that you have the right to contribute the work and that you are offering it under the repository’s license.

If you are not comfortable contributing under those terms, please do not submit the pull request.

## Maintainer discretion

Pull requests may be accepted, revised, deferred, or closed at the maintainer’s discretion.

A contribution being useful does not create an obligation to merge future work, assign paid tasks, or provide compensation.
